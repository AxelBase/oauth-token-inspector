Privacy Policy

Privacy Policy

Effective Date: November 10, 2025

1. Introduction

Welcome to the OAuth Token Inspector, a **100% client-side**, open-source web tool developed by Axelbase. This Privacy Policy explains how we handle your information when you use our JWT inspection utility. We are **committed to absolute privacy** — your tokens never leave your browser. No data is collected, stored, or transmitted. This policy ensures full transparency and trust in a security-sensitive environment.

2. Information We Do Not Collect

The OAuth Token Inspector operates **entirely in your browser**. We do **not** collect:

  • Personal Data: No names, emails, IP addresses, or device identifiers.
  • Token Content: Any JWT you paste is processed locally using vanilla JavaScript — **never sent to any server**.
  • Usage Analytics: No tracking, cookies, localStorage, or telemetry. No Google Analytics, Mixpanel, or similar tools.
  • Session Data: The app is **stateless** — page reloads reset everything.

Because decoding happens via atob() and JSON.parse() in your browser, **we have zero access** to your input.

3. How the Tool Works

When you use the OAuth Token Inspector:

  • Local Processing: JWT is split, Base64URL-decoded, and parsed using jwt.js — all in memory.
  • No Network Calls: Only external request is Bootstrap 5.3.3 CDN (CSS/JS). No API endpoints, no logging.
  • Copy & Export: Uses navigator.clipboard — — stays on your device.
  • Static Build: Output from npm run build is pure HTML/JS/CSS — deployable offline.

This design guarantees **complete data isolation** — ideal for inspecting production, sandbox, or internal tokens.

4. Third-Party Services

The only external dependency is **Bootstrap CDN** (cdn.jsdelivr.net) for styling and dark mode. This CDN does **not** track users or set cookies in this context. No ads, no analytics, no trackers. All core logic is bundled locally via Vite.

5. Cookies and Tracking

No cookies are used. No localStorage persistence. No fingerprinting. Your session ends when you close the tab. The tool is **fully stateless** and **privacy-by-design**.

6. Data Security

While we don’t handle data, we recommend:

  • Use HTTPS (GitHub Pages enforces it)
  • Avoid pasting highly sensitive live tokens
  • Clear browser cache if needed

Security of your device and network is your responsibility.

7. Children's Privacy

This tool is not directed at children under 13. We do not knowingly collect data from minors.

8. International Compliance

Since **no data is collected**, we comply by default with GDPR, POPIA, CCPA, and global privacy laws.

9. Changes to This Policy

Updates will be posted here with a new effective date. Last updated: **November 10, 2025**.

10. Contact

Questions? Open an issue on GitHub or reach out to Axelbase on X.

Your privacy is non-negotiable. No data in = no data out. Use with confidence.
