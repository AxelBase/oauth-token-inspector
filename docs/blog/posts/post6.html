Dark Mode & Accessibility

Dark Mode & Accessibility: Built for Developers

Published: November 10, 2025

Developer tools should be fast, secure, and usable. The OAuth Token Inspector was designed with modern workflows in mind: late-night debugging sessions, keyboard-only navigation, and screen reader compatibility — all in a lightweight, static package.

Seamless Dark Mode

Click the sun/moon toggle in the header to switch between light and dark themes. Powered by Bootstrap’s data-bs-theme attribute:

  • Instant switching — no page reload
  • Respects system preference via prefers-color-scheme
  • Full Bootstrap integration — cards, buttons, alerts adapt automatically

Reduce eye strain during long OAuth integration sessions.

Under the Hood

The toggle sets document.documentElement.setAttribute('data-bs-theme', 'dark'). All colors use CSS variables, ensuring consistency across components.

Accessibility First (WCAG 2.1 AA)

Every interactive element is:

  • Keyboard navigable — Tab to textarea, Parse, Clear, Copy, Theme
  • Focus visible — Bootstrap’s :focus styles
  • Screen reader friendly — ARIA labels, proper heading hierarchy
  • High contrast — passes Lighthouse audits in both themes

Tested With

  • NVDA (Windows)
  • VoiceOver (macOS)
  • ChromeVox
  • Android TalkBack

Responsive Design

Works on:

  • Desktop (≥1920px)
  • Tablet
  • Mobile (≥320px)

Paste tokens on your phone, toggle theme, copy JSON — all with touch-friendly controls.

Why This Matters

Inclusive tools improve productivity for all developers. A tool that works at 2 AM with dark mode and voice control is a tool that gets used.

FAQ

Does dark mode persist?

No — stateless design. Refreshes to system preference. Matches SRS NFR-03.

Can I force light mode?

Yes. Click the toggle or disable prefers-color-scheme in dev tools.

Is it printable?

Yes. Print styles remove backgrounds, keep text readable.
